DESIGN PRINCIPLES

Built on three hard commitments.

⚖

Governance-first

Every action logged with a reason. Asks before touching important code. Reviews its own work after every two tasks. Compliance score: 0 to 100. You don't have to watch it — it watches itself.

◈

Self-solving architecture

Searches the web when it doesn't know. Reviews failed sessions and adjusts. Extracts working patterns into reusable rules. Gets better the more you use it.

◉

Human-in-control

You set the autonomy level — fully autonomous, approval-required, or advisory-only. Pause mid-task without losing state. Stalled agents get killed and restarted automatically.

THE CREW

Seven specialized agents.
Each owns a lane.

Named after physicists. Each is a specialist, not a generalist. Role boundaries are enforced by governance — a reviewer can't write code, a scout can't make edits.

ARLINE

Scout

Reconnaissance
Maps the codebase before any work begins. Identifies patterns, entities, dependencies, and data flow. Produces the intelligence that all other agents rely on. Cannot edit source files — read-only by governance rule.
WHEELER

Planner

Decomposition
Converts scout findings and requirements into executable task graphs with dependencies, verification criteria, and file boundaries. Uses goal-backward planning — starts from the desired outcome, works back to the first step.
DYSON

Worker

Implementation
The only agent allowed to write source code. Operates under strict TDD — RED → GREEN → refactor. Deep Execution Standard: no partial reports, no stubs. 100% completion or escalate.
MURRAY

Reviewer

Adversarial Review
Assumes all worker output is subtly wrong until proven otherwise. Multi-technique rotation across type tracing, state mutation, spec contradiction, and security scans. Cannot fix — only flags.
HANS

Verifier

Invariant Analysis
Traces data flow and checks invariant preservation. Verifies that worker implementation matches the approved data model. Vacuous test detection — deletes the implementation and checks if tests still pass.
DIRAC

Auditor

Claim Verification
Checks that what agents claim matches ground truth. Every assertion needs evidence — file:line citations, test output, direct observation. Trust-but-verify at the orchestration layer.
TUKEY

Researcher

Web Research
Live web search via Perplexity — no stale training data. Multi-angle research across 2–4 queries per topic. Synthesizes findings into actionable intelligence before planning starts.
EXTENDED CREW

+5 Specialists

Conditional Dispatch
Oracle (architecture decisions), Metis (plan consulting), Momus (adversarial plan review), Pauli (governance critic), Noether (library research). Dispatched conditionally based on task complexity.

INSTALLATION
